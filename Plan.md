# Plan - Cycle 7: 미완료 스트라이크 프레임 원자성 확인 테스트

## 목표
스트라이크 프레임이 보너스 투구 부족으로 미완료일 때, `total`에 부분 값이 섞여 들어가지
않고(atomic), 해당 프레임과 그 이후 프레임이 통째로 제외되는지 확인한다.
(예: 8번 프레임 스트라이크 후 9번 프레임도 스트라이크로 던지고 게임이 거기서 멈춘 경우
— 8번 프레임 보너스가 1개만 존재하고 1개는 없어서 8번, 9번 프레임 모두 미확정으로 제외되어야 함)

## 테스트
- 파일: `test_game.py`
- `test_unresolved_strike_excludes_frame_and_later`: 프레임1~7 오픈(1,1×7=14투구) +
  8번 프레임 스트라이크(10) + 9번 프레임 스트라이크(10) 후 종료(총 16투구)
  - 기대 점수: 14 (프레임1~7만 반영, 8·9번 프레임은 보너스 부족으로 통째로 제외)

## 접근 방식
- 새 프로덕션 코드 작성 전, 기존 Cycle 6 구현(`try/except IndexError`)으로 통과하는지 먼저 실행
- 통과하면: 확인 테스트로 커밋 (Cycle 6 구현이 이 엣지 케이스를 원자적으로 처리함을 검증, 코드 변경 없음)
- 실패하면: 원인 분석 후 Plan 갱신, 최소 코드로 수정
