# Plan - Cycle 3: 스트라이크 보너스

## 목표
프레임 첫 투구가 10(스트라이크)이면 해당 프레임은 투구 1회로 종료.
프레임 점수 = 10 + 다음 두 투구의 핀 수 합. 스트라이크 프레임은 투구를 1개만 소비하므로
다음 프레임 시작 인덱스는 1칸만 이동해야 한다 (기존 스페어/오픈은 2칸 이동 유지).

## 테스트
- 파일: `test_game.py`
- `test_strike_bonus`: 첫 프레임 스트라이크(10) + 다음 투구 3, 4 (2번째 프레임) + 나머지 16회 거터(0)
  - 기대 점수: (10+3+4) + (3+4) + 0*8 = 24
- `test_strike_in_ninth_frame`: 프레임1~8 거터(16회 0) + 9번째 프레임 스트라이크(10) + 10번째 프레임 6, 2
  - 기대 점수: 0*8 + (10+6+2) + (6+2) = 26
  - 스트라이크가 마지막(10번째)이 아닌 프레임에서도 인덱스 이동이 올바른지 확인 (9번째 프레임 이후 10번째 프레임으로 정상 진입)

## 접근 방식
- `score()` 프레임 순회에서 각 프레임 시작 시 `rolls[i] == 10`이면 스트라이크 처리:
  `10 + rolls[i+1] + rolls[i+2]`를 프레임 점수로 더하고 `i += 1`
- 스트라이크 아니면 기존 스페어/오픈 로직 그대로, `i += 2`
